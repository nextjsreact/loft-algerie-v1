🔧 Corrections des Traductions - Méthodes de Paiement

✅ Problèmes Identifiés et Corrigés :

  • Chaînes codées en dur : Remplacées par des clés de traduction
  • Types de méthodes de paiement : Ajout des traductions manquantes
  • Textes d'interface : Uniformisation des traductions
  • Fonction de traduction : Ajout d'une fonction pour traduire les types

🔧 Corrections Appliquées :

1. Ajout des clés de traduction manquantes :

  • bankTransfer → "Virement Bancaire" / "Bank Transfer" / "تحويل بنكي"
  • creditCard → "Carte de Crédit" / "Credit Card" / "بطاقة ائتمان"
  • cash → "Espèces" / "Cash" / "نقد"
  • additionalInfoAvailable → "Informations supplémentaires disponibles"
  • noAdditionalDetails → "Aucun détail supplémentaire"
  • active → "Actif" / "Active" / "نشط"
  • modify → "Modifier" / "Modify" / "تعديل"

2. Remplacement des chaînes codées en dur :

  • ❌ "Informations supplémentaires disponibles"
  • ✅ {t('additionalInfoAvailable')}
  • ❌ "Aucun détail supplémentaire"
  • ✅ {t('noAdditionalDetails')}
  • ❌ "Actif"
  • ✅ {t('active')}
  • ❌ "Modifier"
  • ✅ {t('modify')}

3. Fonction de traduction des types :

const getPaymentMethodTypeTranslation = (type: string, t: any) => {
  switch (type?.toLowerCase()) {
    case 'card':
    case 'credit_card':
    case 'debit_card':
      return t('creditCard')
    case 'cash':
    case 'especes':
      return t('cash')
    case 'bank':
    case 'bank_transfer':
      return t('bankTransfer')
    case 'paypal':
      return t('paypal')
    default:
      return type
  }
}
